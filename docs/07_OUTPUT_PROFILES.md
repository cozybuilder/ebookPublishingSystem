# Ebook Publishing System — 출력 프로파일 정의 v0.1

이 문서는 "하나의 원고(AST)로부터 어떤 결과물을 어떤 규칙으로 생산할 것인가"를 정의한다.
출력물은 **기능이 아니라 프로파일(Output Profile)** 이다.

> 본 단계는 **출력 프로파일 정의**다.
> 디자인 시스템 / TypeScript 타입 / JSON 스키마 / 파서 / PDF·DOCX 생성 / 렌더러 / 템플릿 엔진은 포함하지 않는다.

관련 문서: [입력 언어](01_MARKDOWN_RULES.md) · [페이지 타입](02_PAGE_TYPES.md) · [컴포넌트](03_COMPONENTS.md) · [파이프라인](04_PIPELINE.md) · [출력 전략](05_OUTPUT_STRATEGY.md) · [AST 스키마](06_AST_SCHEMA.md)

---

## 핵심 설계 원칙

1. **단일 AST 공유** : 모든 출력 프로파일은 **동일한 AST**(06 문서의 단일 원본)를 사용한다.
2. **원고 재작성 없음** : 프로파일마다 원고를 다시 쓰지 않는다.
3. **선택적 사용** : 프로파일은 AST에서 **필요한 정보만 선택**해 쓴다(전체 또는 발췌).
4. **프로파일 추가만으로 확장** : 새로운 출력물 = 새 프로파일 1개 선언. 파이프라인/원고/AST는 그대로 재사용.
5. **출력물은 기능이 아니라 선언이다** : 코드 분기가 아니라 "프로파일 명세"로 표현된다.

---

## 출력 프로파일이란

출력 프로파일은 다음 **3요소의 조합**으로 정의된다.

| 요소 | 의미 |
| --- | --- |
| **출력 포맷** | 결과물의 형식. PDF / DOCX / 이미지. |
| **페이지 선택 규칙** | AST에서 무엇을 포함/제외할지. 전체 / 앞부분 N / 특정 블록 타입만 / 발췌. |
| **레이아웃 변형** | 어떻게 배치/구성할지. 고정형(인쇄) / 흐름형(편집) / 이미지 캔버스. |

> 같은 AST라도 (포맷 × 선택 × 레이아웃) 조합이 달라지면 다른 출력물이 된다.

## 프로파일 정의

### 1. FullBookPDF

- **목적** : 완성된 전자책 본편 배포.
- **대상 사용자** : 구매한 독자(최종 소비자).
- **출력 포맷** : PDF.
- **사용 데이터** : AST 전체(Metadata + 모든 Chapter + 모든 Block).
- **포함 페이지** : CoverPage, CopyrightPage, TableOfContentsPage, ChapterPage(전체), ContentPage(전체) 및 강조 페이지 일체.
- **제외 페이지** : 없음(전 페이지).
- **레이아웃 특징** : 고정형. 인쇄/뷰어 품질 우선. 다른 프로파일의 기준이 되는 가장 완전한 결과물.

### 2. EditableDOCX

- **목적** : 편집·재가공 가능한 문서 버전 제공.
- **대상 사용자** : 저자/편집자(내부), 편집본을 원하는 구매자.
- **출력 포맷** : DOCX.
- **사용 데이터** : AST 전체.
- **포함 페이지** : 본편과 동일 구성(Cover/Copyright/TOC/Chapter/Content 전체) — 단, 지면이 아닌 흐름형 문서로.
- **제외 페이지** : 없음(전 내용). 단, 인쇄 전용 고정 장식은 단순화될 수 있음.
- **레이아웃 특징** : 흐름형. 편집 편의 우선. 카드형 강조 요소는 워드 표/스타일로 근사. 디자인 충실도보다 수정 가능성 중시.

### 3. KmongPreviewPDF

- **목적** : 판매 페이지에서 구매 전 일부를 보여주는 미리보기.
- **대상 사용자** : 구매를 고민하는 잠재 구매자.
- **출력 포맷** : PDF.
- **사용 데이터** : AST의 **앞부분 일부**(Metadata + Chapter 1의 일부 Block).
- **포함 페이지** : CoverPage, TableOfContentsPage, Chapter 1 일부(ChapterPage + ContentPage 수 페이지), "이후 내용은 본편에서" 안내 페이지.
- **제외 페이지** : CopyrightPage(선택), Chapter 1 후반 이후 전 챕터.
- **레이아웃 특징** : FullBookPDF와 동일한 고정형 스타일을 쓰되 **페이지 수 제한**. 구매 전환이 목적이라 핵심 가치만 맛보임.

### 4. DetailPageImages

- **목적** : 크몽 상품 상세 페이지를 채우는 홍보/설명 이미지.
- **대상 사용자** : 상품 페이지 방문자(구매 검토 단계).
- **출력 포맷** : 이미지(PNG 등, 세로로 긴 상세 배너).
- **사용 데이터** : Metadata(제목/부제/저자), 표지 이미지(image type: cover), 대표 블록 발췌(TOC, 주요 checklist/result 등).
- **포함 페이지** : 별도 지면 아님 — **이미지 산출물**. 표지 기반 키비주얼 + 핵심 포인트 카드 이미지들.
- **제외 페이지** : 본문 전체 흐름(전문 노출 안 함).
- **레이아웃 특징** : 이미지 캔버스. 세로 긴 배너 구성. 마케팅 카피 강조, 시각 임팩트 우선.

### 5. SNSPromoImages

- **목적** : SNS 채널에서 책을 홍보하는 정사각/스토리형 이미지.
- **대상 사용자** : SNS 피드/스토리 노출 대상(잠재 독자, 팔로워).
- **출력 포맷** : 이미지(정사각 1:1, 스토리 9:16 등 규격별).
- **사용 데이터** : Metadata(제목/부제), 표지 이미지, 한 문장 요약(result 블록 등) 발췌.
- **포함 페이지** : 별도 지면 아님 — 표지·핵심 문구 기반 **단일/소수 이미지**.
- **제외 페이지** : 본문 대부분(후킹용 최소 정보만).
- **레이아웃 특징** : 이미지 캔버스. 플랫폼 규격별 변형. 후킹 카피 + 표지 비주얼 중심, 정보보다 시선 유도.

### 6. ChecklistPDF

- **목적** : 본문의 체크리스트만 모은 실전 부록 / 리드 마그넷.
- **대상 사용자** : 실행을 원하는 독자, 무료 배포용 미끼 자료 수령자.
- **출력 포맷** : PDF.
- **사용 데이터** : AST에서 **`checklist` 블록만 수집**(+ 책 제목/출처 표시).
- **포함 페이지** : 간단한 표지 + ChecklistPage(들).
- **제외 페이지** : 본문 단락, 그 외 모든 블록(checklist 외 전부).
- **레이아웃 특징** : 단일 주제(실행 점검)에 집중한 경량 PDF. 인쇄/체크 친화적. 본편과 별개 단독 배포 가능.

## 프로파일 요약표

| 프로파일 | 포맷 | 페이지 선택 규칙 | 레이아웃 변형 | 주 사용자 |
| --- | --- | --- | --- | --- |
| FullBookPDF | PDF | 전체 | 고정형 | 구매 독자 |
| EditableDOCX | DOCX | 전체 | 흐름형 | 저자/편집자 |
| KmongPreviewPDF | PDF | 앞부분 일부(Ch.1 일부) | 고정형(페이지 제한) | 잠재 구매자 |
| DetailPageImages | 이미지 | 발췌(표지+대표 블록) | 이미지 캔버스(세로 배너) | 상품 페이지 방문자 |
| SNSPromoImages | 이미지 | 핵심 발췌(표지+요약) | 이미지 캔버스(규격별) | SNS 노출 대상 |
| ChecklistPDF | PDF | checklist 블록만 | 경량 고정형 | 실행형 독자 |

## 프로파일과 파이프라인의 관계

- [1]~[3] (분석 → 블록 추출 → AST) : **모든 프로파일 공통**. 1회 수행한 AST를 공유.
- [4] Page 구성 : 프로파일의 **페이지 선택 규칙**이 적용되는 지점.
- [5] Component 매핑 : 공통(블록 → 컴포넌트). 발췌 시 선택된 블록만 매핑.
- [6] Layout 적용 : 프로파일의 **레이아웃 변형**이 적용되는 지점.
- [7] Output : 프로파일의 **출력 포맷**으로 내보내는 지점.

→ 신규 출력물 추가 = 새 프로파일(3요소) 선언. [1]~[3]과 원고/AST는 손대지 않는다.
