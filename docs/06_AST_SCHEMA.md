# Ebook Publishing System — AST 스키마 정의 v0.1

이 문서는 Ebook Publishing System의 **단일 원본 구조(AST: 책 문서 모델)** 를 정의한다.
파이프라인 [3] 단계가 산출하며, 이후 모든 출력물이 공유하는 기준이다.

> 모든 출력물(PDF · DOCX · 크몽 미리보기 · 상세페이지 이미지 · SNS 홍보 이미지 · 체크리스트 PDF)은
> **동일한 단일 원본 구조**를 사용한다. 출력물별 차이는 이 구조를 어떻게 선택/배치/스타일하느냐일 뿐이다.

> 본 단계는 **구조의 의미 정의**다.
> TypeScript 인터페이스 / JSON 스키마 / 파서 / 렌더러 / PDF·DOCX / 템플릿은 포함하지 않는다.
> 객체가 가져야 할 정보를 **자연어 수준**으로만 기술한다.

관련 문서: [입력 언어](01_MARKDOWN_RULES.md) · [페이지 타입](02_PAGE_TYPES.md) · [컴포넌트](03_COMPONENTS.md) · [파이프라인](04_PIPELINE.md) · [출력 전략](05_OUTPUT_STRATEGY.md)

## 핵심 설계 원칙

1. **AST는 디자인을 포함하지 않는다.** 폰트·색·여백·카드 모양 등 어떤 시각 정보도 갖지 않는다.
2. **AST는 PDF를 알지 못한다.** 페이지 크기, 인쇄, 뷰어 개념이 없다.
3. **AST는 DOCX를 알지 못한다.** 워드 스타일·문단 서식 개념이 없다.
4. **AST는 오직 책의 구조와 의미만 표현한다.** "무엇인지"와 "무엇을 담는지"만 안다.
5. **스타일과 레이아웃은 이후 단계의 책임이다.** AST는 어떤 출력물에도 종속되지 않는 중립 구조다.
6. **순서 보존.** 원고의 등장 순서가 AST 안에서 그대로 유지된다.

## 객체 정의

### Book (책)

- 단일 원본의 최상위 객체. 책 한 권 전체를 표현한다.
- 가져야 할 정보:
  - **메타데이터(Metadata)** : 책의 기본 정보 1개.
  - **챕터 목록(Chapter[])** : 책을 구성하는 챕터들. 등장 순서대로 보존.
- 디자인/출력 관련 정보는 갖지 않는다. (출력물 선택은 상위의 출력 프로파일이 담당)

### Metadata (메타데이터)

- 책의 식별·표시에 필요한 기본 정보.
- 가져야 할 정보:
  - **제목(title)** : `#` 으로 입력된 책 제목.
  - **부제(subtitle)** : `subtitle:` 값. 없을 수 있음.
  - **저자(author)** : `author:` 값. 없을 수 있음.
- (판권/발행 정보 등 추가 필드는 향후 확장 가능. 단, 표시용 의미 정보에 한함.)

### Chapter (챕터)

- 책을 구성하는 하나의 장. `## Chapter N.` 으로 시작된 구간.
- 가져야 할 정보:
  - **번호(number)** : 챕터 순번.
  - **제목(title)** : 챕터 제목.
  - **블록 목록(Block[])** : 챕터 본문을 이루는 블록들. 등장 순서대로 보존.
- 챕터는 자신에 속한 블록만 안다. 페이지 개념은 없다.

### Block (블록)

- 챕터 안의 의미 단위. 본문 한 단락이거나 특수 시각 요소다.
- 공통으로 가져야 할 정보:
  - **종류(type)** : 아래 "Block 종류" 중 하나.
  - **내용(content)** : 종류별로 형태가 다른 의미 데이터(자연어 텍스트, 항목 목록, 키-값 등).
- 블록은 자신이 "무엇인지"와 "무슨 내용을 담는지"만 안다.
  어떻게 보일지(카드/색/배치)는 알지 못한다.

### Page (페이지)

- **주의: Page는 AST에 속하지 않는다.** 페이지는 파이프라인 [4] 단계에서
  AST(Book→Chapter→Block)를 출력 목적에 맞게 묶을 때 생성되는 **하위(파생) 구조**다.
- 가져야 할 정보(파생 단계 기준):
  - **페이지 타입** : CoverPage / ChapterPage / ContentPage 등.
  - **담긴 블록 참조** : 이 페이지가 표현하는 AST 블록들(또는 메타데이터)에 대한 참조.
- Page는 출력물마다 다르게 구성될 수 있다(전체/발췌). 따라서 단일 원본(AST)이 아니라
  출력별 산출물로 분리해 둔다. → 단일 원본은 어떤 출력에도 종속되지 않는다.

## Block 종류 (현재 지원 11종)

| 종류 | 의미 | 담는 내용(자연어) |
| --- | --- | --- |
| paragraph | 일반 본문 단락 | 텍스트 한 단락 |
| table | 정보 정리용 표 | 열 제목들 + 행 데이터들 (표준 Markdown 표) |
| checklist | 체크리스트 | 점검 항목들의 목록 |
| compare | 비교표(카드형) | 열 제목들 + 행 데이터들 |
| before-after | 전후 비교 | before 내용, after 내용 |
| prompt | 프롬프트 박스 | 프롬프트 텍스트(여러 줄 가능) |
| steps | 단계 설명 | 순서가 있는 단계 항목 목록 |
| faq | 자주 묻는 질문 | 질문/답변 쌍들의 목록 |
| warning | 주의 박스 | 주의 텍스트 |
| result | 결과 요약 | 핵심 요약 텍스트 |
| image | 이미지 슬롯 | 식별자(id), 이미지 유형(type), 생성 설명(prompt) |

> 입력 언어(01)의 9개 Container Block + 표준 본문 단락(paragraph) + 표준 Markdown 표(table) = 11종.
> 제목/부제/저자는 블록이 아니라 **Metadata** 에 속한다.
>
> **table vs compare** : `table`은 정보 정리용 일반 표(표준 Markdown 표 문법으로 입력),
> `compare`는 대상/선택지를 강조 비교하는 카드형 요소(`:::compare` 블록). 의미가 다른 별개 블록이다.

## 구조 한눈에 보기

```text
Book
├─ Metadata (title, subtitle?, author?)
└─ Chapter[]            (number, title)
   └─ Block[]           (type, content)   ← 등장 순서 보존
        type ∈ { paragraph, table, checklist, compare, before-after,
                 prompt, steps, faq, warning, result, image }

※ Page 는 AST의 일부가 아니라, 출력 단계에서 AST로부터 파생되는 구조다.
```

## 단일 원본으로서의 의미

- AST는 **출력물 6종 모두의 공통 입력**이다.
- 전체형 출력(PDF·DOCX)은 AST 전체를 사용한다.
- 발췌형 출력(미리보기·상세이미지·SNS·체크리스트)은 AST에서 일부(특정 챕터·특정 블록 종류)를 골라 쓴다.
  - 예: 체크리스트 PDF = 모든 Chapter의 `checklist` 블록만 수집.
- 따라서 AST는 **블록 종류 단위·챕터 단위로 접근·필터링하기 쉬운 형태**여야 한다(발췌형의 요구사항).
