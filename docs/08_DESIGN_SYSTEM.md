## 0. 디자인 철학

- 전자책은 교재가 아니라 **상품**이다. 독자는 글과 이미지·디자인을 함께 소비한다.
- 목표는 두꺼운 책이 아니라 **끝까지 읽히는 책**이다.
- 디자인은 **일관성 > 화려함**. 책마다 달라 보이면 안 된다 — 같은 출판사의 책처럼 보여야 한다.
- 모든 규칙은 **재사용 가능한 토큰**으로 정의한다(색·간격·타이포 단계). 책마다 값을 새로 정하지 않는다.

## 1. 브랜드 컬러

기본 팔레트는 **Navy · Orange · Cyan** 3색 + 중립색(검정/회색/흰색)으로 구성한다.

| 컬러 | 역할 | 사용 위치 |
| --- | --- | --- |
| **Navy** | 기본/구조 | 본문 텍스트, 헤더, 챕터 제목, 표 헤더선 등 책의 뼈대. 신뢰감·안정감. |
| **Orange** | 강조/행동 | 핵심 키워드, 강조 문구, WarningCard·ResultCard 강조, CTA 성격 요소. 시선 유도. |
| **Cyan** | 보조/정보 | 보조 정보, 캡션 강조, PromptCard·StepsCard 등 정보형 카드의 보조 톤. 차분한 액센트. |

### 컬러 사용 원칙

- 한 지면에서 **강조색(Orange)은 절제**해서 쓴다 — 강조가 많으면 강조가 사라진다.
- Navy = 기본, Orange = 강조, Cyan = 보조. **이 위계를 모든 책에서 동일하게** 적용한다.
- 중립색(흰 배경 / 회색 보조선 / 검정에 가까운 Navy 텍스트)으로 여백과 가독성을 확보한다.
- 컬러 의미는 컴포넌트에 일관 매핑한다(예: warning 계열은 항상 Orange 톤, 정보 계열은 Cyan 톤).

## 2. 타이포그래피

원칙: **가독성 우선.** 장식보다 읽힘을 먼저 본다. 위계(크기·굵기)로 구조를 전달한다.

| 요소 | 역할 | 스타일 방향 |
| --- | --- | --- |
| 책 제목 | 표지 최상위 위계 | 가장 크고 굵게. Navy(또는 표지 이미지 대비색). 강한 존재감. |
| 챕터 제목 | 챕터 시작 구분 | 본문보다 크고 굵게. Navy. 챕터 표지에서 시각적 휴지(休止) 역할. |
| 본문 | 읽기의 기본 단위 | 충분한 크기·줄간격. Navy 계열. 장식 최소화, 긴 호흡으로 읽히게. |
| 캡션 | 이미지/표 보조 설명 | 본문보다 작게. 회색 또는 Cyan 톤. 보조 정보임을 명확히. |
| 강조 문구 | 핵심 메시지 부각 | 본문 내 Orange 강조 또는 굵게. 절제해서 사용. |

### 타이포 원칙

- 크기 단계(스케일)는 **고정된 단계 집합**에서만 고른다(제목 > 챕터 > 본문 > 캡션). 임의 크기 금지.
- 줄간격은 본문 가독성을 최우선으로 넉넉하게.
- 한글 가독성을 기준으로 폰트를 선택한다(구체 폰트는 형식화 단계에서 확정).

## 3. 간격 시스템

원칙: 간격은 **고정된 단계(spacing scale)** 에서만 사용한다. 임의 여백 금지 → 책마다 일관.

| 요소 | 기본 규칙 |
| --- | --- |
| 페이지 여백 | 사방에 넉넉한 마진. 본문이 가장자리에 붙지 않게. 인쇄/뷰어 모두 안정적인 안쪽 여백 확보. |
| 챕터 간격 | 챕터 사이는 가장 큰 간격(또는 챕터 표지로 분리). 시각적 휴지를 크게. |
| 문단 간격 | 문단 사이는 한 단계 간격. 줄바꿈만으로 붙이지 않는다. |
| 카드 내부 여백 | 카드 테두리와 내용 사이 일정한 패딩. 모든 카드가 동일 패딩 단계를 공유. |

- 간격 위계: **페이지 여백 ≥ 챕터 간격 > 카드 내부 여백 ≥ 문단 간격** (상대 관계 고정).

## 4. 카드 스타일 (공통 원칙 우선)

대상: ChecklistCard · TableCard · CompareCard · BeforeAfterCard · PromptCard · StepsCard · FAQCard · WarningCard · ResultCard.

> 개별 카드 디자인보다 **공통 시각 규칙을 먼저** 정의한다. 모든 카드는 같은 골격을 공유하고,
> 의미(강조/정보/중립)에 따라 컬러 톤만 달라진다.

### 4.1 공통 골격

- 모든 카드는 **동일한 모서리 처리·테두리/배경 방식·내부 패딩 단계**를 공유한다.
- 카드 상단에 **라벨/아이콘 영역**(카드 종류 표시)을 둘 수 있다 — 위치·크기 규칙 공통.
- 카드 폭은 본문 컬럼 폭에 맞춘다. 카드 간 세로 간격은 간격 시스템을 따른다.

### 4.2 의미 기반 컬러 톤 (3군)

| 톤 군 | 해당 카드 | 컬러 방향 |
| --- | --- | --- |
| 강조형 | WarningCard, ResultCard | Orange 톤(주의·결론 부각) |
| 정보형 | PromptCard, StepsCard, FAQCard | Cyan 톤(보조·안내) |
| 중립형 | ChecklistCard, TableCard, CompareCard | Navy/중립 톤(구조·정리) |

### 4.3 카드별 보조 규칙

- ChecklistCard : 체크박스(☐) + 항목. 중립 톤.
- TableCard : 헤더 행 강조(Navy 선) + 데이터 행. 정보 정리에 충실, 장식 최소.
- CompareCard : 대상별 열/카드 구분을 시각적으로 분리. 차이를 부각(중립 톤 + 선택적 강조).
- BeforeAfterCard : 좌(before)/우(after) 대비. 변화 방향을 색·배치로 암시.
- PromptCard : 모노 느낌의 복사 친화 박스. Cyan 톤 액센트.
- StepsCard : 자동 번호(1·2·3) 강조. Cyan 톤.
- FAQCard : 질문 강조(굵게) + 답변 들여쓰기. Cyan 톤.
- WarningCard : Orange 강조 + 주의 아이콘.
- ResultCard : Orange 강조 박스. 핵심 결론 각인.

## 5. 이미지 규칙

원칙: **AI 이미지는 최소한으로** 사용한다. 정보 전달 요소는 시스템이 자동 생성한다.

### 5.1 AI 이미지 우선 사용 위치 (제한적)

- 표지 (cover)
- 챕터 대표 이미지 (chapter)
- 크몽 썸네일 (thumbnail)

### 5.2 시스템 자동 생성 요소 (AI 이미지 아님)

- 표, 체크리스트, 플로우, 비교표, 카드형 요약, FAQ, 주의 박스, 프롬프트 카드, 그래프 등.
- 이들은 **컴포넌트/디자인 시스템으로 렌더링**한다 → 책마다 품질이 일정하고, 수정·재생산이 쉽다.

### 5.3 이유

- 정보 요소를 AI 이미지로 만들면 일관성이 깨지고 수정이 어렵다.
- 시스템 생성 요소는 텍스트 기반이라 검색·편집·재출력·번역에 유리하다.

## 6. 이미지 캔버스 규격 (이미지 출력 프로파일)

대상 프로파일: **DetailPageImages**, **SNSPromoImages**. (구체 픽셀값은 미정 — 비율만 정의.)

| 규격 | 비율 | 주 용도 |
| --- | --- | --- |
| 정사각 | 1:1 | SNS 피드(SNSPromoImages 기본) |
| 스토리/세로 | 9:16 | SNS 스토리·릴스(SNSPromoImages) |
| 세로 긴 배너 | 세로 길이 가변(폭 고정, 긴 세로) | 크몽 상세 페이지(DetailPageImages) |

### 캔버스 원칙

- 모든 이미지 캔버스도 **동일 브랜드 컬러·타이포·여백 규칙**을 따른다(책과 같은 시각 언어).
- DetailPageImages : 세로 긴 배너에 표지 키비주얼 + 핵심 포인트 카드 이미지를 위→아래로 배치.
- SNSPromoImages : 후킹 카피 + 표지 비주얼 중심. 규격(1:1 / 9:16)별 변형.
- 캔버스 안에서도 강조(Orange)는 절제, 가독성 우선 원칙 동일 적용.

## 설계 원칙 요약

1. **일관성 우선** : 색·타이포·간격을 고정 토큰 단계로 두어 책마다 같은 품질.
2. **의미 기반 매핑** : 컬러/카드 톤은 의미(강조·정보·중립)에 고정 매핑.
3. **AI 이미지 최소화** : 표지·챕터·썸네일만 AI, 정보 요소는 시스템 생성.
4. **하나의 시각 언어** : 책 지면과 이미지 캔버스(상세·SNS)가 같은 디자인 시스템을 공유.
