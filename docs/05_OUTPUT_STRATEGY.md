# Ebook Publishing System — 출력 전략 정의 v0.1

이 문서는 Ebook Publishing System이 **하나의 원고(`book.md`)로부터 생산하는 출력물들**과
그 전략을 정의한다.

> Ebook Publishing System은 PDF 생성기가 아니라 **출판 시스템**이다.
> 목적은 전자책 한 권 제작이 아니라, 하나의 원고로 여러 결과물을 **지속적으로 생산**하는 것이다.

> 본 단계는 **출력 전략 정의**다.
> AST 스키마 / 디자인 시스템 / 파서 / PDF·DOCX 생성 / 렌더러 / 템플릿 엔진은 포함하지 않는다.

관련 문서: [입력 언어](01_MARKDOWN_RULES.md) · [페이지 타입](02_PAGE_TYPES.md) · [컴포넌트](03_COMPONENTS.md) · [파이프라인](04_PIPELINE.md)

---

## 공통 원칙

1. **단일 원본(Single Source)** : 모든 출력물은 하나의 원고(`book.md` → 중간 데이터 구조)를 단일 원본으로 사용한다.
2. **원고 재작성 없음** : 새로운 출력 포맷이 추가되어도 원고를 다시 작성하지 않는다.
3. **출력물별 차이는 선택과 배치뿐** : 출력물마다 달라지는 것은
   "어떤 페이지를 포함할지(선택)"와 "어떻게 배치/스타일할지(레이아웃)"뿐이다.
4. **파이프라인 재사용** : [1]~[6] 단계(분석~레이아웃)는 공유하고, 출력물별 차이는 [7] Output 단계에서 처리한다.
5. **확장 지향** : 출력물은 계속 늘어날 수 있다. 새 출력물 추가 = 새 "출력 프로파일" 추가이지, 시스템 재설계가 아니다.

## 출력물 정의

### 1. PDF (전자책 본편)

- **목적** : 완성된 전자책 본편을 배포 가능한 형태로 제공.
- **대상 사용자** : 구매한 독자(최종 소비자).
- **원본 데이터** : 중간 데이터 구조 전체(메타데이터 + 모든 챕터 + 모든 블록).
- **필요한 페이지** : CoverPage, CopyrightPage, TableOfContentsPage,
  ChapterPage(전체), ContentPage(전체), 및 강조 페이지 일체. **전 페이지 포함.**
- **특징** : 가장 완전한 결과물. 고정 레이아웃, 인쇄/뷰어 품질 우선. 다른 출력물의 기준이 됨.

### 2. DOCX (편집용 문서)

- **목적** : 편집·수정이 가능한 문서 버전 제공(교정, 재가공, 협업).
- **대상 사용자** : 저자/편집자(내부), 또는 편집본을 원하는 구매자.
- **원본 데이터** : 중간 데이터 구조 전체.
- **필요한 페이지** : PDF와 동일한 전체 구성(단, 고정 레이아웃이 아닌 흐름형 문서).
- **특징** : 편집 가능성 우선. 카드형 강조 요소는 워드 표/스타일로 근사. 디자인 충실도보다 수정 편의성 중시.

### 3. 크몽 미리보기 PDF

- **목적** : 판매 페이지에서 구매 전 일부를 보여주는 미리보기 제공.
- **대상 사용자** : 구매를 고민하는 잠재 구매자.
- **원본 데이터** : 중간 데이터 구조에서 **앞부분 일부만 선택**.
- **필요한 페이지** : CoverPage, TableOfContentsPage, Chapter 1 일부(ChapterPage + ContentPage 수 페이지),
  필요 시 "이후 내용은 본편에서" 안내 페이지. **부분 포함.**
- **특징** : 구매 전환이 목적. 핵심 가치를 맛보이되 전체를 노출하지 않음. 페이지 수 제한.

### 4. 상세페이지 이미지

- **목적** : 크몽 상품 상세 페이지를 채우는 홍보/설명 이미지.
- **대상 사용자** : 상품 페이지 방문자(구매 검토 단계).
- **원본 데이터** : 메타데이터(제목/부제/저자), 표지 이미지, 대표 블록(목차, 주요 checklist/result 등) 발췌.
- **필요한 페이지** : 별도 지면이 아닌 **이미지 산출물**. 표지 기반 키비주얼 + 핵심 포인트 카드 이미지들.
- **특징** : PDF 페이지가 아니라 이미지 포맷(PNG 등). 세로로 긴 상세 배너 형태가 일반적. 마케팅 카피 강조.

### 5. SNS 홍보 이미지

- **목적** : SNS 채널에서 책을 홍보하는 정사각/스토리형 이미지.
- **대상 사용자** : SNS 피드/스토리 노출 대상(잠재 독자, 팔로워).
- **원본 데이터** : 제목/부제, 표지 이미지, 한 문장 요약(result 블록 등) 발췌.
- **필요한 페이지** : 별도 지면 아님. 표지·핵심 문구 기반 **단일/소수 이미지**.
- **특징** : 플랫폼별 규격(정사각 1:1, 스토리 9:16 등). 시선을 끄는 카피 + 표지 비주얼 중심. 정보보다 후킹.

### 6. 체크리스트 PDF

- **목적** : 본문 속 체크리스트만 모아 실전용 부록/리드 마그넷으로 제공.
- **대상 사용자** : 실행을 원하는 독자, 또는 무료 배포용 미끼 자료 수령자.
- **원본 데이터** : 중간 데이터 구조에서 **`checklist` 블록만 수집**(+ 제목/출처 표시).
- **필요한 페이지** : 간단한 표지 + ChecklistPage(들). **체크리스트 한정.**
- **특징** : 단일 주제(실행 점검)에 집중한 경량 PDF. 인쇄/체크 친화적. 본편과 별개로 단독 배포 가능.

| 출력물 | 포맷 | 포함 범위 | 주 사용자 | 핵심 목적 |
| --- | --- | --- | --- | --- |
| PDF 본편 | PDF | 전체 | 구매 독자 | 완성된 전자책 배포 |
| DOCX | DOCX | 전체(흐름형) | 저자/편집자 | 편집·재가공 |

## 출력 프로파일 개념

각 출력물은 하나의 **출력 프로파일(Output Profile)** 로 표현할 수 있다.
프로파일은 다음을 선언한다(구현 스키마는 이후 단계에서 확정).

- **포맷** : PDF / DOCX / 이미지
- **페이지 선택 규칙** : 전체 / 앞부분 N / 특정 블록 타입만 / 발췌
- **레이아웃 변형** : 고정형(PDF) / 흐름형(DOCX) / 이미지 캔버스(SNS·상세)

→ 출력물 추가 = 새 프로파일 선언. 1~6단계 파이프라인과 원고는 그대로 재사용한다.
