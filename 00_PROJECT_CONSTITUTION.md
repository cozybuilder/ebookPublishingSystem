# Ebook Publishing System 기준정보 v0.1

## 프로젝트 목적

Markdown 원고를 입력하면 CozyBuilder Lab 스타일의 전자책 상품 패키지를 자동 생성하는 출판 엔진을 만든다.

목표는 전자책 한 권 제작이 아니라, 전자책을 지속적으로 생산할 수 있는 시스템 구축이다.

## 핵심 입력

* book.md

## 핵심 출력

* PDF
* DOCX
* 체크리스트
* 인포그래픽
* 표
* Before / After
* 판권 페이지
* 크몽용 미리보기 PDF
* 상세페이지 이미지

## 철학

전자책은 교재가 아니다.
전자책은 상품이다.

사람들은 글만 읽지 않는다.
글과 이미지, 디자인을 함께 소비한다.

목표는 두꺼운 책이 아니라 끝까지 읽히는 책이다.

## 역할

### 코지

CEO
방향 결정, 최종 승인

### 코비

PM / 디자인 디렉터
시스템 설계, UI/UX, 전자책 구조, 디자인 규칙, 이미지 프롬프트 규칙

### 클로

개발자
PDF 생성, DOCX 생성, 레이아웃 엔진, 템플릿 엔진 구현

## 핵심 기능

1. Markdown 입력
2. 페이지 타입 자동 인식
3. 텍스트의 시각 요소 변환
4. 챕터 시작 페이지 자동 생성
5. 이미지 슬롯 추출
6. CozyBuilder Lab 디자인 시스템 적용
7. 컴포넌트 자동 생성
8. PDF / DOCX / 크몽 미리보기 / 상세페이지 이미지 출력

## 이미지 원칙

모든 이미지를 AI로 만들지 않는다.

AI 이미지는 다음에만 우선 사용한다.

* 표지
* 챕터 대표 이미지
* 크몽 썸네일

다음 요소는 시스템이 자동 생성한다.

* 표
* 체크리스트
* 그래프
* 플로우
* 비교표
* 카드형 요약
* FAQ
* 주의 박스
* 프롬프트 카드

## 최종 흐름

코지: 1권 제작 지시
클로: 원고 작성
Ebook Publishing System: 자동 디자인
출력: PDF
코지: 출간
