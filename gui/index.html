전자책 만들기

AI로 만든 원고와 이미지를 넣으면
PDF · DOCX · EPUB 전자책이 완성됩니다.

사용법 — 전자책 만들기 (STEP 1~9)
STEP 1. AI로 원고 만들기 — 전자책 주제를 입력하면 AI 원고 작성 프롬프트가 완성됩니다. 복사해 ChatGPT·Claude에 붙여넣어 Markdown 원고(.md)를 받으세요.
STEP 2. 프로젝트 선택 — "+ 새 프로젝트 만들기"로 책 하나를 만들고, 받은 원고를 [원고 불러오기]로 가져옵니다. (하나의 책 = 하나의 프로젝트)
STEP 3. 출판 정보 설정 — 저자명·발행자·저작권자·저자 소개·면책 문구를 입력합니다. (책 제목·부제는 원고에서 자동, 발행일은 오늘 자동)
STEP 4. 미리보기 생성 — 원고만으로 HTML 미리보기를 만들어 구조·레이아웃을 먼저 확인합니다.
STEP 5. 이미지 만들기 — 표지·본문 이미지의 프롬프트를 복사해 외부 AI로 생성합니다. 표지는 cover.png, 본문은 IMG-001.png 형식.
STEP 6. 이미지 넣기 — 프로젝트 images 폴더에 이미지를 넣고, 미리보기를 다시 만들어 반영을 확인합니다.
STEP 7. 최종 확인 — 표지·목차·이미지·누락 여부와 출력 형식을 점검합니다.
STEP 8. 전자책 만들기 — PDF·출판용 PDF·DOCX·EPUB·HTML과 크몽 자료를 한 번에 생성합니다.
STEP 9. 결과 확인 — 완성된 파일을 열거나 프로젝트 output 폴더를 엽니다.

전자책 만들기

아래 단계를 차례대로 따라가면 원고에서 PDF · DOCX · EPUB 전자책까지 완성됩니다.

1. AI로 원고 만들기

전자책 주제 하나만 입력하면 됩니다. 아래 프롬프트에 자동으로 들어가니, 복사해서 AI(ChatGPT · Claude 등)에게 보내세요. 원고와 이미지 프롬프트까지 함께 만들어집니다.

AI 원고 작성 프롬프트

위에 입력한 주제가 아래 프롬프트에 자동으로 반영됩니다. [복사] 후 AI에게 붙여넣으세요.
전자책 주제:
여기에 주제 입력

당신은 전문 전자책 작가입니다.
위 주제로 실제 판매 가능한 전자책 원고를 작성해주세요.

조건:
1. Markdown 형식으로 작성
2. 읽기 쉬운 문체
3. 초보자도 이해할 수 있도록 설명
4. 최소 8개 이상의 Chapter 구성
5. 각 Chapter는 충분한 내용을 포함 (일반 본문 위주)
6. 컴포넌트(표·체크리스트·FAQ·Timeline 등)는 꼭 필요할 때만, 한 Chapter에 1~2개 이내로만 사용
7. 억지로 여러 컴포넌트를 넣지 말 것 — 목표는 "읽기 편한 책"이지 "컴포넌트 쇼케이스"가 아님
8. 에세이·명언집·자기계발서 등은 컴포넌트를 거의 쓰지 말고 본문 흐름으로 작성
9. 중요한 부분은 강조, 실전 팁 포함, 독자가 바로 따라할 수 있도록 구성
10. 이미지가 필요한 부분은 이미지 블록 형식으로 작성

[정확한 형식 — 이 형식을 반드시 지켜주세요]
- 제목: 첫 줄에  # 책 제목
- 부제/저자: 그 아래에  subtitle: 부제   /   author: 저자
- 각 장(章): "## Chapter 1. 제목" 형식 (숫자와 점 필수, 1부터 순서대로)
- 강조: ==중요한 문장==
- 컴포넌트 예 (꼭 필요할 때만 사용 — 형식 참고용):
:::checklist
- 항목 1
- 항목 2
:::

:::faq
q: 질문
a: 답변
:::

- 표지 이미지는 원고 최상단(제목·부제·저자 아래, 첫 Chapter 앞)에 반드시 1개:
:::image
id: cover
type: cover
filename: cover.png
prompt: 전자책 표지 디자인. 책 제목·부제·저자명이 들어갈 여백을 둔 세로형 A4 비율(1240x1748 이상) 표지. 주제에 맞는 분위기·색감을 상세히.
:::

- 본문 이미지는 IMG-001 부터, 각 형식 그대로:
:::image
id: IMG-001
type: chapter
filename: IMG-001.png
prompt: 본문과 어울리는 이미지 설명(생성 AI가 바로 쓸 수 있게 상세히)
:::

이미지 블록 규칙:
- 표지: id=cover, type=cover, filename=cover.png — 딱 1개, 최상단. 절대 IMG-001 로 만들지 말 것.
- 본문: id=IMG-001, IMG-002, IMG-003 … 순서, type=chapter, filename=IMG-00N.png
- 표지는 세로 A4(1240x1748 이상), 본문은 9:16 세로(1080x1920 이상) 권장
- prompt는 생성 AI가 바로 쓸 수 있게 상세하게

목표는 실제 돈을 받고 판매할 수 있는 수준의 전자책입니다.
💡 이미지는 나중에 따로 생각하는 것이 아닙니다. AI가 원고를 쓸 때 이미지 프롬프트까지 함께 준비합니다. 다음 단계에서 그 프롬프트로 이미지를 생성해 넣으면 됩니다.

## Chapter 1. 첫 번째 장 제목

여기에 문단을 씁니다. 중요한 부분은 ==이렇게== 강조할 수 있습니다.

:::checklist

- 두 번째 할 일
:::

## Chapter 2. 두 번째 장 제목

다음 장의 내용을 이어서 씁니다.

2. 프로젝트 선택

하나의 책 = 하나의 프로젝트입니다. 프로젝트마다 원고·이미지·결과물이 따로 보관되어 다른 책과 섞이지 않습니다.

프로젝트를 불러오는 중…

3. 출판 정보 설정

표지·판권에 들어갈 저작권·발행 정보를 입력하세요. 저자명·발행자·저작권자·저자 소개·면책 문구만 설정합니다.

먼저 2단계에서 프로젝트를 선택하세요.

4. 미리보기 생성

먼저 원고만으로 미리보기(HTML)를 만들어 구조와 레이아웃을 확인합니다. 이미지가 아직 없으면 자리표시자로 보입니다.

원고 구조가 의도대로 나오는지 먼저 확인하세요. 문제가 있으면 1단계 규칙을 참고해 원고를 고친 뒤 다시 만들면 됩니다.

5. 이미지 만들기

원고에 필요한 이미지 목록입니다. 각 프롬프트를 복사해 외부 AI 이미지 도구에서 만들고, 표시된 파일명으로 저장하세요.

6. 이미지 넣기 & 다시 미리보기

이 프로젝트의 images 폴더에 이미지를 직접 넣으세요. 표지는 cover.png, 본문 이미지는 IMG-001.png 형식으로 저장합니다.

7. 최종 확인

전자책을 만들기 전에 마지막으로 점검합니다.

생성될 출력 형식

• book.pdf — 대표 전자책 PDF
• book-paged.pdf — 출판용 PDF(쪽번호·러닝헤드·목차 쪽번호)
• book.docx · book.epub · book.html + 크몽 자료(미리보기·상세 이미지)

8. 전자책 만들기

모든 형식을 한 번에 생성합니다. 이미지가 많으면 1~2분 정도 걸릴 수 있습니다.

9. 결과 확인

완성된 파일을 열어 확인하세요.

결과물은 output/ 폴더에 보관됩니다. (개발용 npm test 실행 시에만 비워지니, 배포 파일은 여기서 바로 확인·복사하세요.)
